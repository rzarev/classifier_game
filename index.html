Classify!

A model fitting game

Rumen Zarev

Complex models and small sample sizes

One of the main problems in data analysis and machine learning is the balance between the need for a model to be flexible enough to fit a given data pattern, but not too flexible that it overfits. This is especially important for small sample sizes.

A more complex model or a smaller data set increase the discrepancy between in-sample and out-of-sample model performance.

Demonstration

Here is what happens in a simple regression context, when we inlude many extraneous features.

set.seed(135)
x <- runif(50, -1, 1)
y <- 0.2 * x + runif(50, -1, 1)
df <- data.frame(x = x, "x^2" = x^2, "x^3" = x^3, "sin(x)" = sin(x), y = y)

model <- lm(y ~ ., data = df)
coef(model)

## (Intercept)           x         x.2         x.3      sin.x. 
##   0.1184225 -84.8951845  -0.0231573  12.8838298  85.4065880

The game

The "Classify!" illustrates this problem by presenting the player with a sample from simulated data. The data is two-dimensional, of two classes.

The possible models allowed involve a linear cutoff function, over 16 derived features---monomials in \(x\) and \(y\) up to \(x^3y^3\).

The player can either pick coefficients directly or use penalized logistic regression. To control the complexity of the model the player can choose to include only a subset of the features, as well as control the penalty parameter \(\lambda\).
