Classify!

A model fitting game

Rumen Zarev

Complex models and small sample sizes

One of the main problems in data analysis and machine learning is the balance between the need for a model to be flexible enough to fit a given data pattern, but not too flexible that it overfits. This is especially important for small sample sizes.

A more complex model or a smaller data set increase the discrepancy between in-sample and out-of-sample model performance.

Demonstration

To demonstrate, we take a classification example where the data is perfectly separable. We train a linear classifier (using logistic regression) using the three relevant variables, and again using all nine variables. We train both models on the first 50 points, and test on the remaining 50.

set.seed(135)
x <- runif(100, -1, 1); y <- runif(100, -1, 1); z <- runif(100, -1, 1)
p <- runif(100, -1, 1); q <- runif(100, -1, 1); r <- runif(100, -1, 1)
t <- runif(100, -1, 1); u <- runif(100, -1, 1); v <- runif(100, -1, 1)
outcome <- as.integer(x + 0.5 * y + 2 * z > 0)

##   Variables In sample Out of sample
## 1         3      100%           98%
## 2         9      100%           94%

We see that while both models perform perfectly in-sample, neither does so out-of-sample, and the model with 9 variables does significantly worse.

The game

The "Classify!" illustrates this problem by presenting the player with a sample from simulated data. The data is two-dimensional, of two classes.

The possible models allowed involve a linear cutoff function, over 16 derived features---monomials in \(x\) and \(y\) up to \(x^3y^3\).

The player can either pick coefficients directly or use penalized logistic regression. To control the complexity of the model the player can choose to include only a subset of the features, as well as control the penalty parameter \(\lambda\).

Validation

To simulate the real world effect of in-sample vs out-of-sample error differences, the end result evaluation is given by testing the player's final hypothesis on a held out validation set.

To prevent cheating, we only allow looking at the out-of-sample result once, after which a new game is started with a new data set.
